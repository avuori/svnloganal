FAQ

Q: How do I use this program?
A: svnloganal --help

Q: I have problems compiling the program.
A: Ensure that the libxml2 headers are found from /usr/include/libxml2/libxml.
I used gcc (GCC) 4.1.2 to compile it.

Q: Where can I report bugs to?
A: Check my homepage for contact info.

Q: Why is it written in C? These programs are written in Python!
A: I wanted to write something in C, anything, and I needed this kind of software.

Q: I have a great feature suggestion! Is this project still active?
A: I probably don't need that feature myself, meaning, you have to make it yourself.

Q: Why does it not understand the xml-formatted subversion log data but only reads flat, human-readable logs?
A: I was excessively lazy not to read svn help log before coding the parser. Now I don't need that feature – but you can do it!

Q: Does the version numbering follow any logic?
A: They are simply revision numbers. (Update: I have migrated to git so now I'm just increasing the number every time I make a release.)

Q: Why doesn't this work on French (or any non-English) locale?
A: Because the adhoc parser is assuming English logs. Use an English locale when running svn log: e.g.: LC_ALL=en_US svn -v log | ./svnloganal.
